Postman podstawy

O szkoleniu:

Chcesz zdobyć podstawową wiedzę o narzędziu Postman? Zastanawiasz się, jak się do tego zabrać? Sprawdź, czy to szkolenie jest dla Ciebie!

Cel szkolenia:

Celem szkolenia jest zapoznanie uczestników z jednym z najpopularniejszych narzędzi do testowania API.

Program szkolenia:

  • Wprowadzenie do Postman:
    • Instalacja Postman
    • Omówienie interfejsu użytkownika Postman
  • Podstawy REST API:
    • Działanie protokołu HTTP
    • Żądania i odpowiedzi HTTP
    • Nagłówki w żądaniach i odpowiedziach
    • Podstawowe metody HTTP
  • Dodatkowe narzędzia: Swagger, konsola deweloperska
  • Tworzenie zapytań do API:
    • Kolekcje
    • Pierwsze zapytania: GET, POST, PUT, DELETE
    • Parametry ścieżki vs parametry zapytania
    • Zmienne i środowiska
  • Pisanie testów i skryptów:
    • Pierwszy test API
    • Statusy kodów odpowiedzi
    • Weryfikacja treści odpowiedzi
    • Dynamiczne zmienne
    • Przekazywanie danych między zapytaniami
    • Skrypty żądań wstępnych
  • Autentykacja (autoryzacja HTTP):
    • Omówienie różnych typów autentykacji
    • Praktyczne przykłady wybranych typów autoryzacji
  • Uruchamianie testów z linii poleceń:
    • Postman CLI
    • Newman

Przed szkoleniem:

Dla kogo dedykowane jest szkolenie?

Szkolenie przeznaczone jest dla testerów oprogramowania, którzy chcieliby poznać narzędzie do wykorzystania w codziennej pracy. Jest także skierowane do osób bez doświadczenia w testowaniu API i pracy z narzędziem.

Jak się przygotować ?

Uczestnicy szkolenia nie muszą posiadać specjalistycznej ani programistycznej wiedzy. Program zakłada naukę testowania API
od podstaw. Szkolenie zakłada korzystanie z własnego sprzętu komputerowego, a środowisko zostanie przygotowane
i udostępnione przez szkoleniowca.

Zalety szkolenia:

Samodzielność: Wykorzystywanie zdobytej wiedzy i umiejętności do samodzielnego tworzenia testów API.
Podstawy programowania: Szkolenie może obejmować naukę podstaw języka programowania, takiego jak JavaScript.
Ta umiejętność pozwoli na pisanie bardziej zaawansowanych skryptów i dostosowanie testów API do konkretnych potrzeb.
Wiedza praktyczna: Uczestnicy będą mieli okazję do natychmiastowego stosowania zdobytej wiedzy w swoich projektach, co zapewnia płynne przeniesienie wiedzy na praktykę.
